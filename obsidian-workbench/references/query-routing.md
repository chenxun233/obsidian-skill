# Query Routing

Use the wiki in this order.

1. Search compiled note families directly using the user's terms, likely aliases, and root concepts.
2. Read the relevant concept, source-tied, flow, deep, recipe, issue-record, and relationship pages.
3. Follow strong wikilinks from the best-matching pages to complete the local graph.
4. Check `Wiki Ops/lint.md` if the question may hit stale or disputed material.
5. Read the current query day file under `Query/YYYY-MM/YYYY-MM-DD.md` when recent wording or context matters; this is also the default log target for the new query.
6. Read at most a few recent older query day files under `Query/YYYY-MM/` when the answer may have been asked recently but not today.
7. If a prior query entry already answers the same question well enough, point to it as answer history, but still ground the new answer in compiled pages.
8. Read `Wiki Ops/logs/` only when the user asks for operation history or when the answer depends on how the wiki changed over time.

Do not treat query logs or Wiki Ops logs as the primary knowledge source. The compiled pages are primary. Query has no default learning-checklist layer; reusable query results should become durable compiled notes directly.
