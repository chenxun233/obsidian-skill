# Source Modes

Choose the ingest unit from how the source enters the wiki.

- `Whole-source ingest`
 Use for one article, one paper, one standalone document, or one full video session.
 Expected result: one learning checklist that lets the student master the source-derived skill or concept step by step.

- `Source-unit ingest`
 Use for one chapter, one episode segment, one meeting, or one bounded section inside a larger source.
 Expected result: add or revise checklist ticks for that source unit, preserving local provenance and a one-step-at-a-time learning order.

- `Incremental source continuation`
 Use when the source already exists in the wiki and a new section arrives later.
 Expected result: revise the existing checklist first, add new ticks in dependency order, and update reusable notes only when the current tick needs them.

Branch only when branching improves understanding. Do not create one note per heading by default.

The checklist is not raw-resource compression. It should answer: "What sequence should a student follow to master this skill or concept, and how can they check that mastery?" A checklist may be deduced from a resource, but it should stand alone enough that the learner can use it as the mastery-check source without reopening the original material. If it comes from ingest, keep an explicit source-resource link for provenance.

Checklist granularity follows source shape. One episode or short article usually maps to one checklist. A book, long course, or large investigation may map to multiple checklists, such as one per chapter or major branch, when that makes the incremental learning path clearer.
