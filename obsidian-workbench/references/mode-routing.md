# Obsidian Skill Routing

Choose the mode from the main output, not from the file extension alone.

- Simple vault read, list, grep, or one-off file lookup -> vault read/search mode
- Markdown syntax, wikilinks, embeds, callouts, frontmatter, or rendering fixes -> markdown syntax mode
- Running Obsidian app commands, plugin reload, DOM inspection, screenshots, console errors, or theme/plugin debugging -> CLI and plugin-debug mode
- Durable note creation, revision, routing, or note-family choice -> durable note mode
- Raw source, issue record, investigation, article, paper, video, or skill/concept path that should become a learning checklist -> learning checklist ingest mode
- Compiled wiki answer, synthesis across existing notes, contradiction check, or maintenance pass -> compiled wiki query mode
- Obsidian Bases or `.base` work -> bases mode

One task may cross modes, but start from the mode that owns the main output.
