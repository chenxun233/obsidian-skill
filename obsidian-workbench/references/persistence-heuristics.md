# Persistence Heuristics

Persist the result when one or more of these are true:

- The answer creates a durable comparison or decision surface the user is likely to reuse.
- The answer synthesizes multiple pages into one clearer governing root.
- The answer reveals a missing relationship, missing concept page, or missing index path.
- The answer corrects or supersedes an existing compiled claim with clear support.

Do not persist the result when the answer is mainly:

- a one-off clarification
- a narrow retrieval that is already obvious from one page
- temporary operational chatter
- a question whose evidence is still incomplete

When in doubt, answer without creating a new note. Still append the query record to `Query/YYYY-MM/YYYY-MM-DD.md` for ordinary wiki queries. Do not create a query learning-checklist note for ordinary queries.
