# Ingest

Convert raw source material into a reusable story-led learning checklist in the Obsidian vault. Treat the source as immutable. Prefer revising an existing checklist over creating a duplicate.

## What to do

1. Read `references/source-modes.md` to determine how to handle the source type.
2. Read `references/story-led-checklists.md` for checklist structure and story-led principles.
3. Read `references/wiki-ops-conventions.md` for vault conventions.
4. Read `references/review-rubric.md` to self-check the output before delivering.
5. If the source is a PDF, read `references/pdf-ingest-workflow.md` before extracting.
6. Search the vault for an existing checklist on the same topic — revise it rather than duplicating.
7. Build the checklist: story-led, incremental, with checkpoints where the learner needs them.

## Non-negotiable rules

- Do not summarize or paraphrase the source into flat bullet lists. The checklist should follow the causal arc of the source material.
- Each checkpoint should state what the learner should be able to do or explain after completing it — not just what they read.
- Log only checklist **creation** in `Wiki Ops/logs/YYYY-MM/YYYY-MM-DD.md`.
